# 一、系统总体架构设计

智慧农业控制系统采用 **物联网三层架构模型**：

**感知层 → 网络层 → 应用层**

系统总体结构如下：

```
感知层（传感器设备）
    │
    │ 采集环境数据
    ↓
网关层（智能控制主机）
    │
    │ 数据处理 / 指令转发
    ↓
网络通信层（WiFi / MQTT / HTTP）
    │
    │ 数据传输
    ↓
应用层（OpenHarmony APP）
    │
    │ 数据展示 / 控制设备
    ↓
用户
```

系统实现 **数据采集 + 智能控制 + 可视化管理**。

------

# 二、硬件系统架构设计

硬件部分主要包括 **传感器、控制设备和网关**。

### 硬件系统组成

```
环境传感器
    ├─ 温度传感器
    ├─ 湿度传感器
    ├─ 光照传感器
    └─ CO₂传感器

控制设备
    ├─ 水泵
    ├─ 水阀
    ├─ LED补光灯
    └─ 扩展设备（风扇等）

智能网关
    └─ 负责设备通信与数据处理
```

### 硬件系统数据流程

```
传感器采集数据
        ↓
智能网关读取数据
        ↓
数据发送到APP
        ↓
APP展示环境数据
        ↓
用户控制设备
        ↓
APP发送控制指令
        ↓
网关执行设备控制
```

# 三、软件系统架构设计

软件系统采用 **模块化架构设计**。

整体软件结构如下：

```
智慧农业APP
│
├── 用户界面层（UI Layer）
│
├── 业务逻辑层（Business Layer）
│   ├─ 环境监测模块
│   ├─ 智能灌溉模块
│   ├─ 智能光照模块
│   ├─ 设备控制模块
│   ├─ 预警模块
│   └─ 扩展设备模块
│
├── 数据处理层（Data Layer）
│   ├─ 数据解析
│   ├─ 数据存储
│   └─ 数据计算
│
└── 通信层（Communication Layer）
    ├─ 设备通信
    └─ 网络通信
```

### 各层作用

#### 1 UI层

负责：

- 数据可视化
- 用户操作界面

例如：

- 环境监测界面
- 设备控制界面
- 报警界面

------

#### 2 业务逻辑层

负责：

- 灌溉逻辑
- 光照控制
- 设备控制
- 报警判断

------

#### 3 数据处理层

负责：

- 传感器数据解析
- 数据存储
- 数据统计

------

#### 4 通信层

负责：

- APP与网关通信
- 指令发送
- 数据接收

------

# 四、系统功能模块设计

系统共设计 **6个核心功能模块**。

------

# 1 设备管理模块

功能：

- 设备连接管理
- 设备状态查看
- 设备控制

子功能：

```
设备列表
设备添加
设备删除
设备状态监控
设备控制
```

------

# 2 环境监测模块

功能：

实时采集并展示环境数据。

采集数据：

- 温度
- 湿度
- 光照
- CO₂

功能：

```
实时数据展示
历史数据记录
数据图表展示
```

------

# 3 智能灌溉模块

功能：

- 自动灌溉
- 手动灌溉
- 灌溉记录

自动灌溉逻辑：

```
读取土壤湿度
      ↓
湿度低于阈值
      ↓
计算灌溉水量
      ↓
启动水泵
      ↓
灌溉完成
```

------

# 4 智能光照模块

功能：

- 光照监测
- LED控制
- 自动补光

控制逻辑：

```
检测光照
     ↓
光照低于阈值
     ↓
开启LED
     ↓
调节亮度
```

------

# 5 预警模块

功能：

- 阈值设置
- 异常检测
- 报警提示

例如：

```
温度 > 35℃
湿度 < 30%
CO₂异常
```

系统自动报警。

------

# 6 扩展设备模块

为了验证系统扩展能力，可以增加设备。

例如：

**通风风扇**

功能：

- 温度过高时自动开启
- 用户可手动控制

# 六、接口设计

系统需要设计 **APP与设备通信接口**。

------

## 1 获取环境数据接口

功能：

获取传感器数据。

示例：

```
GET /sensor/data
```

返回：

```
{
 "temperature": 26,
 "humidity": 60,
 "light": 350,
 "co2": 450
}
```

------

## 2 控制水泵接口

```
POST /device/waterPump
```

参数：

```
{
 "status":"on"
}
```

------

## 3 LED控制接口

```
POST /device/led
```

参数：

```
{
 "brightness":80
}
```

------

## 4 设置阈值接口

```
POST /threshold/set
```

参数：

```
{
 "temperature_max":35,
 "humidity_min":30
}
```

# 七、数据库设计

系统数据库主要存储 **设备信息、环境数据、报警信息等**。

主要设计 **4张核心表**。

------

# 1 设备表 device

字段：

| 字段        | 类型    | 说明     |
| ----------- | ------- | -------- |
| device_id   | int     | 设备ID   |
| device_name | varchar | 设备名称 |
| device_type | varchar | 设备类型 |
| status      | int     | 设备状态 |

------

# 2 传感器数据表 sensor_data

| 字段        | 类型     |
| ----------- | -------- |
| id          | int      |
| temperature | float    |
| humidity    | float    |
| light       | float    |
| co2         | float    |
| time        | datetime |

------

# 3 灌溉记录表 irrigation_log

| 字段         | 类型     |
| ------------ | -------- |
| id           | int      |
| water_amount | float    |
| start_time   | datetime |
| end_time     | datetime |

------

# 4 报警记录表 alarm

| 字段        | 类型     |
| ----------- | -------- |
| id          | int      |
| alarm_type  | varchar  |
| alarm_value | float    |
| time        | datetime |
