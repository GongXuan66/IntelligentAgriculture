## 当前系统分析

### 现有逻辑

```
触发条件: 土壤湿度 < 阈值 (默认40%)
用水量计算: 用水量 = 湿度差 × 风险系数
风险系数: 基于温度/湿度/光照的简单线性调整
```

### 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| FLOW_RATE_L_PER_SEC | 0.5 升/秒 | 水泵流量 |
| SOIL_MOISTURE_GAIN_PER_LITER | 1.6667 % | 每升水提升的土壤湿度 |
| HUMIDITY_GAIN_PER_LITER | 0.3 % | 每升水提升的空气湿度 |
| MAX_SOIL_MOISTURE_GAIN | 35 % | 单次灌溉最大土壤湿度提升 |

### 存在的问题

1. **被动响应式**：等土壤干涸才灌溉，已造成水分胁迫
2. **固定参数**：不考虑土壤类型、作物差异
3. **无预测能力**：无法预判未来湿度变化
4. **单目标决策**：仅考虑湿度，忽略成本、效率等因素

## 方向一：预测性灌溉

### 核心原理

通过历史数据预测未来土壤湿度变化趋势，提前做出灌溉决策，而不是被动响应。

```
传统策略: 湿度 < 40% → 触发灌溉 → 已造成水分胁迫

预测策略: 
  预测2小时后湿度 < 35% 
  → 现在就开始灌溉（在蒸发高峰前补水）
  → 或等到傍晚灌溉（减少蒸发损失）
```

### 技术架构

```
数据输入:
├── 历史土壤湿度序列（过去24小时）
├── 历史环境数据（温度、光照、湿度）
├── 天气预报数据（未来温度、降雨概率）
└── 时间特征（小时、季节）

预测模型:
├── 简单方案: 指数加权移动平均 (EWMA)
├── 中等方案: ARIMA 时序预测
└── 高级方案: LSTM/Transformer 深度学习

输出:
├── 未来2-6小时湿度预测值
├── 湿度变化趋势（上升/下降/稳定）
└── 最佳灌溉时间窗口
```

### 简单实现方案：EWMA趋势预测

```java
/**
 * 指数加权移动平均预测
 * 适用于湿度趋势预测，计算量小，易于实现
 */
public class MoisturePredictor {
    
    // 衰减因子，越大越重视近期数据
    private static final double ALPHA = 0.3;
    
    /**
     * 预测未来N小时后的湿度
     * @param history 过去N个时间点的湿度数据
     * @param hoursAhead 预测未来几小时
     * @return 预测湿度值
     */
    public double predict(List<Double> history, int hoursAhead) {
        if (history == null || history.isEmpty()) {
            return -1;
        }
        
        // 计算EWMA平滑值
        double ewma = history.get(0);
        for (int i = 1; i < history.size(); i++) {
            ewma = ALPHA * history.get(i) + (1 - ALPHA) * ewma;
        }
        
        // 计算趋势（最近几个点的平均变化率）
        double trend = 0;
        int trendWindow = Math.min(3, history.size() - 1);
        for (int i = history.size() - trendWindow; i < history.size(); i++) {
            trend += history.get(i) - history.get(i - 1);
        }
        trend /= trendWindow;
        
        // 预测未来值
        return ewma + trend * hoursAhead;
    }
    
    /**
     * 判断是否需要预防性灌溉
     */
    public PredictResult analyze(List<Double> history, double threshold) {
        double current = history.get(history.size() - 1);
        double predict2h = predict(history, 2);
        double predict4h = predict(history, 4);
        
        PredictResult result = new PredictResult();
        result.setCurrentMoisture(current);
        result.setPredict2h(predict2h);
        result.setPredict4h(predict4h);
        
        // 下降趋势且预测值低于阈值
        if (predict2h < threshold && current >= threshold) {
            result.setNeedPreIrrigation(true);
            result.setReason("预测2小时后湿度将降至" + String.format("%.1f", predict2h) + "%，建议提前灌溉");
        }
        
        return result;
    }
}
```

### 优势与挑战

**优势：**
- 节水：避免高温时段灌溉的高蒸发损失
- 护苗：预防性补水，植物不会经历水分胁迫
- 智能感强：展示系统"预判"能力

**挑战：**
- 需要足够的历史数据
- 预测准确度依赖数据质量
- 天气API依赖

## 方向二：自适应学习算法

### 核心原理

系统不是用固定参数，而是根据每次灌溉的实际效果自动调整计算模型。

```
当前问题:
参数 SOIL_MOISTURE_GAIN_PER_LITER = 1.67%
但这是估计值，不同土壤、温度下实际效果不同

自适应学习:
┌─────────────────────────────────────────┐
│  灌溉前: 土壤湿度 30%                      │
│  灌溉量: 10升水                           │
│  灌溉后: 土壤湿度 44%                      │
│  实际效果: 每升提升 1.4%（非1.67%）         │
│  → 更新参数为加权平均                      │
│  新参数 = 0.8×旧值 + 0.2×实测值            │
└─────────────────────────────────────────┘
```

### 学习算法实现

```java
/**
 * 自适应学习服务
 */
@Service
public class IrrigationLearningService {
    
    @Autowired
    private IrrigationLearningParamsMapper paramsMapper;
    
    /**
     * 学习并更新参数
     * 在每次灌溉完成后调用
     */
    public void learn(IrrigationLog log) {
        if (log.getSoilMoistureAfter() == null || log.getSoilMoistureBefore() == null) {
            return;
        }
        
        // 计算实际效果
        double actualGain = log.getSoilMoistureAfter() - log.getSoilMoistureBefore();
        double waterAmount = log.getWaterAmount();
        
        if (waterAmount <= 0 || actualGain <= 0) {
            return;
        }
        
        // 计算实际的"每升水提升湿度"
        double actualGainPerLiter = actualGain / waterAmount;
        
        // 获取当前参数
        IrrigationLearningParams param = paramsMapper.findByPointAndName(
            log.getPointId(), "MOISTURE_GAIN_PER_LITER"
        );
        
        if (param == null) {
            // 首次创建
            param = new IrrigationLearningParams();
            param.setPointId(log.getPointId());
            param.setParamName("MOISTURE_GAIN_PER_LITER");
            param.setParamValue(BigDecimal.valueOf(actualGainPerLiter));
            param.setSampleCount(1);
            param.setConfidence(BigDecimal.valueOf(0.3));
            paramsMapper.insert(param);
        } else {
            // 增量学习：加权平均
            int newCount = param.getSampleCount() + 1;
            double oldValue = param.getParamValue().doubleValue();
            
            // 学习率随样本数递减
            double learningRate = 1.0 / Math.min(newCount, 10);
            double newValue = oldValue * (1 - learningRate) + actualGainPerLiter * learningRate;
            
            // 更新置信度
            double newConfidence = Math.min(0.95, 0.3 + newCount * 0.05);
            
            param.setParamValue(BigDecimal.valueOf(newValue));
            param.setSampleCount(newCount);
            param.setConfidence(BigDecimal.valueOf(newConfidence));
            paramsMapper.updateById(param);
        }
    }
}
```

### 优势与挑战

**优势：**
- 越用越准：系统随时间自我优化
- 适应性强：自动适应不同地块、季节
- 实现相对简单：增量学习，无需复杂模型

**挑战：**
- 需要记录灌溉前后数据
- 初期数据少时效果不明显

## 方向三：多目标优化

### 核心原理

灌溉不是单一目标，而是需要在多个相互冲突的目标间找到最优解。

### 目标函数设计

```
目标1: 湿度达标率最大化
目标2: 用水量最小化
目标3: 成本优化（峰谷电价）
目标4: 作物健康（避免水分波动）
```

### 优势与挑战

**优势：**
- 全面考虑：不唯节水论，平衡多目标
- 经济性强：考虑电价、人工等成本

**挑战：**
- 算法复杂度高
- 需要明确各目标权重

## 方向四：作物生长阶段感知

### 核心原理

不同作物在不同生长阶段对水分的需求差异很大，灌溉策略应该随生长周期动态调整。

### 作物需水模型

```
以番茄为例:

播种期 (0-7天): 湿度目标 60-70%, 高频少量
幼苗期 (7-30天): 湿度目标 50-60%, 适度控水
开花期 (30-50天): 湿度目标 45-55%, 精准控制
结果期 (50-90天): 湿度目标 55-65%, 需水量最大
成熟期 (90天+): 湿度目标 45-55%, 适当减少
```

**优势：**
- 科学精准：基于农学知识
- 展示效果好：界面可展示"当前作物状态"

**挑战：**
- 需要建立作物知识库
- 依赖准确的播种日期记录

### Python训练脚本示例

```python
# ml-models/train_lstm.py
import torch
import torch.nn as nn
import numpy as np

class MoistureLSTM(nn.Module):
    """土壤湿度预测LSTM模型"""
    
    def __init__(self, input_size=5, hidden_size=64, num_layers=2):
        super().__init__()
        self.lstm = nn.LSTM(input_size, hidden_size, num_layers, batch_first=True)
        self.fc = nn.Linear(hidden_size, 1)
    
    def forward(self, x):
        out, _ = self.lstm(x)
        out = self.fc(out[:, -1, :])  # 取最后一个时间步
        return out

def train_model(train_data, epochs=100):
    """训练模型"""
    model = MoistureLSTM()
    criterion = nn.MSELoss()
    optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
    
    for epoch in range(epochs):
        for batch_x, batch_y in train_data:
            optimizer.zero_grad()
            output = model(batch_x)
            loss = criterion(output, batch_y)
            loss.backward()
            optimizer.step()
    
    return model

def export_to_onnx(model, save_path):
    """导出为ONNX格式"""
    dummy_input = torch.randn(1, 24, 5)  # batch=1, seq=24, features=5
    torch.onnx.export(model, dummy_input, save_path,
                      input_names=['input'],
                      output_names=['output'],
                      dynamic_axes={'input': {0: 'batch_size'}})

if __name__ == '__main__':
    # 训练并导出
    model = train_model(train_data)
    export_to_onnx(model, 'moisture_lstm.onnx')
    print("模型已导出到 moisture_lstm.onnx")
```

### Java ONNX推理

```java
// 添加Maven依赖
// <dependency>
//     <groupId>com.microsoft.onnxruntime</groupId>
//     <artifactId>onnxruntime</artifactId>
//     <version>1.15.0</version>
// </dependency>

/**
 * ONNX模型预测器
 */
public class ONNXPredictor {
    
    private OrtSession session;
    
    public ONNXPredictor(String modelPath) throws OrtException {
        OrtEnvironment env = OrtEnvironment.getEnvironment();
        this.session = env.createSession(modelPath);
    }
    
    /**
     * 预测未来湿度
     * @param history 历史数据 [湿度, 温度, 光照, 空气湿度, 时间特征]
     * @return 预测的湿度值
     */
    public double predict(float[][] history) throws OrtException {
        // 构造输入张量 [1, seq_len, features]
        float[][][] inputData = new float[1][history.length][history[0].length];
        inputData[0] = history;
        
        OnnxTensor input = OnnxTensor.createTensor(
            OrtEnvironment.getEnvironment(), 
            inputData
        );
        
        // 推理
        OrtSession.Result result = session.run(
            Collections.singletonMap("input", input)
        );
        
        // 获取输出
        float[][] output = (float[][]) result.get(0).getValue();
        return output[0][0];
    }
}
```

/**
 * 智能预测服务（支持AI模式和传统模式切换）
 */
@Service
public class SmartPredictionService {
    
    private EWMAPredictor ewmaPredictor = new EWMAPredictor();
    private ONNXPredictor onnxPredictor;
    
    @Value("${irrigation.prediction.mode:ewma}")
    private String predictionMode;
    
    @PostConstruct
    public void init() {
        if ("lstm".equals(predictionMode)) {
            try {
                onnxPredictor = new ONNXPredictor("models/moisture_lstm.onnx");
                log.info("LSTM模型加载成功");
            } catch (Exception e) {
                log.warn("LSTM模型加载失败，回退到EWMA模式");
                predictionMode = "ewma";
            }
        }
    }
    
    /**
     * 预测湿度（自动选择模型）
     */
    public double predict(List<Double> history, int hoursAhead) {
        if ("lstm".equals(predictionMode) && onnxPredictor != null) {
            try {
                return predictWithLSTM(history);
            } catch (Exception e) {
                log.warn("LSTM预测失败，回退到EWMA");
            }
        }
        return ewmaPredictor.predict(history, hoursAhead);
    }
    
    private double predictWithLSTM(List<Double> history) throws OrtException {
        // 转换数据格式并调用ONNX模型
        float[][] inputData = prepareInput(history);
        return onnxPredictor.predict(inputData);
    }
}
