# 智慧农业控制系统 - 开发计划

## 一、项目概述

基于 OpenHarmony 的智慧农业控制 APP，实现远程监测农作物环境的温度、湿度、光照、二氧化碳，控制水阀、水泵、LED等设备。

## 三、职责划分

### 服务器端（30%代码量）

| 功能 | 说明 |
|------|------|
| API接口 | 提供RESTful API供APP调用 |
| 数据存储 | SQLite/MySQL 数据库，存储历史数据 |
| 数据中转 | 接收传感器数据，转发给APP |
| 模拟网关 | 开发阶段模拟传感器数据 |
| 简单逻辑 | 数据验证、基础查询 |

### APP端（70%代码量，重点）

| 功能 | 说明 |
|------|------|
| UI界面 | 环境数据展示、设备控制界面 |
| 业务逻辑 | 预警判断、灌溉计算、联动规则 |
| 本地缓存 | RelationalStore 存储近期数据 |
| 网络通信 | HTTP请求与服务器交互 |

## 四、数据库设计

### 检测点表 (monitor_point)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键，自增 |
| point_id | TEXT | 检测点ID |
| point_name | TEXT | 名称（如：1号大棚） |
| location | TEXT | 位置 |
| crop_type | TEXT | 作物类型 |
| status | INTEGER | 状态(0停用/1启用) |
| created_at | TEXT | 创建时间 |

### 设备表 (device)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| point_id | INTEGER | 所属检测点ID |
| device_id | TEXT | 设备标识 |
| device_name | TEXT | 设备名称 |
| device_type | TEXT | 设备类型(pump/led/fan/valve) |
| status | INTEGER | 状态(0离线/1在线/2工作中) |
| created_at | TEXT | 创建时间 |

### 环境数据表 (sensor_data)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| point_id | INTEGER | 检测点ID |
| temperature | REAL | 温度(℃) |
| humidity | REAL | 湿度(%) |
| light | REAL | 光照(lux) |
| co2 | REAL | CO₂(ppm) |
| soil_moisture | REAL | 土壤湿度(%) |
| recorded_at | TEXT | 记录时间 |

### 灌溉记录表 (irrigation_log)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| point_id | INTEGER | 检测点ID |
| water_amount | REAL | 灌溉水量(L) |
| duration | INTEGER | 持续时间(秒) |
| mode | INTEGER | 模式(0自动/1手动) |
| start_time | TEXT | 开始时间 |
| end_time | TEXT | 结束时间 |

### 报警记录表 (alarm)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| point_id | INTEGER | 检测点ID |
| alarm_type | TEXT | 报警类型 |
| alarm_value | REAL | 报警数值 |
| threshold | REAL | 阈值 |
| status | INTEGER | 状态(0未处理/1已处理) |
| created_at | TEXT | 报警时间 |

## 五、API接口设计

### 环境数据

```
GET  /api/environment          # 获取当前环境数据
GET  /api/environment/history  # 获取历史数据
POST /api/environment          # 上报环境数据（网关用）
```

### 设备管理

```
GET  /api/devices              # 获取设备列表
GET  /api/device/{id}          # 获取设备详情
POST /api/device/control       # 控制设备
GET  /api/device/{id}/status   # 获取设备状态
```

### 灌溉管理

```
GET  /api/irrigation/logs      # 获取灌溉记录
POST /api/irrigation/start     # 开始灌溉
POST /api/irrigation/stop      # 停止灌溉
```

### 报警管理

```
GET  /api/alarms               # 获取报警列表
PUT  /api/alarm/{id}/handle    # 处理报警
```

### 检测点管理

```
GET  /api/points               # 获取检测点列表
POST /api/point                # 添加检测点
PUT  /api/point/{id}           # 更新检测点
```
