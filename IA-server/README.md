# Git 多人协作零基础详细教程
这份教程专为 Git 新手设计，全程用「傻瓜式步骤」讲解，你跟着做就能和我一起协作编写代码，每个操作都标注「为什么做」「怎么做」，不用记复杂概念。

## 一、前期准备（只做一次）
### 1. 安装 Git（电脑首次用必装）
- **Windows**：
  下载地址：https://git-scm.com/download/win
  安装时全部选「默认选项」，一路下一步即可。
- **Mac**：
  打开「终端」，输入 `git --version`，如果提示未安装，按提示安装即可（或用 `brew install git`）。

### 2. 配置个人信息（关联你的 GitHub/GitLab 账号）
打开 Git Bash（Windows）/ 终端（Mac），输入以下命令（把括号里的内容换成你的信息）：
```bash
# 配置用户名（和你的 GitHub 账号名一致）
git config --global user.name "你的用户名"
# 配置邮箱（和你的 GitHub 注册邮箱一致）
git config --global user.email "你的邮箱@xxx.com"
```
**作用**：让 Git 知道是谁提交的代码，协作时能追溯到每个人的修改。

## 二、第一步：把远程仓库拉到本地（仅第一次做）
### 1. 找到仓库地址
我会给你一个远程仓库地址，格式类似：
`https://github.com/gongxusuan6/IntelligentAgriculture.git`

### 2. 选择本地存放代码的文件夹
比如在桌面新建一个文件夹「Agriculture」，右键点击文件夹 → 选择「Git Bash Here」（Windows）/ 直接打开终端进入该文件夹（Mac）。

### 3. 克隆仓库到本地
在弹出的命令窗口输入：
```bash
git clone https://github.com/gongxusuan6/IntelligentAgriculture.git
```
按回车后，等待几秒，文件夹里就会出现仓库的所有代码（这一步是「把远程代码下载到你电脑上」）。

## 三、日常协作开发：核心步骤（每次写代码都按这个流程）
### 步骤1：切换到主分支，拉取最新代码（必做！避免代码冲突）
每次开始写代码前，先确保你的本地代码是最新的（同步我或其他人的修改）：
```bash
# 进入克隆下来的仓库文件夹（如果没进去的话）
cd IntelligentAgriculture

# 切换到主分支（main 是主分支名，有些仓库叫 master）
git checkout main

# 拉取远程主分支的最新代码（把我的修改同步到你电脑）
git pull origin main
```

### 步骤2：创建自己的功能分支（绝对不要直接在 main 分支写代码！）
为了不影响主分支，你需要创建一个「专属分支」来写自己的代码，分支名建议「功能名-你的名字」，比如你要写登录功能，就叫 `login-张三`：
```bash
# 创建并切换到你的分支（一行命令完成）
git checkout -b login-张三
```
**验证**：输入 `git branch`，如果看到 `* login-张三`（* 代表当前分支），说明切换成功。

### 步骤3：在本地写代码
打开这个文件夹里的代码，随便修改/新增文件（比如改一个 `.py` 文件，或新增一个页面），改完保存即可。

### 步骤4：把你的修改提交到本地 Git
这一步是「告诉 Git 你改了哪些内容」，分3小步：
```bash
# 第一步：查看你修改了哪些文件（可选，确认用）
git status
# 会看到红色的文件，就是你改的内容

# 第二步：把所有修改加入「暂存区」（. 代表所有修改，也可以指定文件，比如 git add login.py）
git add .

# 第三步：提交修改到本地仓库（备注必须写清楚你改了什么）
git commit -m "完成登录页面的按钮样式修改"
```
**备注要求**：简单明了，比如「修复登录按钮点击没反应」「新增用户列表页面」，方便我知道你改了什么。

### 步骤5：推送你的分支到远程仓库（让我看到你的代码）
```bash
# 首次推送你的分支（关联远程分支，只需要输一次）
git push -u origin login-张三

# 之后再修改代码，推送到远程时，直接输这行即可：
git push
```
推送成功后，我就能在远程仓库看到你的分支和代码了。

### 步骤6：同步我的修改（如果我改了主分支代码）
如果你写代码的过程中，我往主分支提交了新代码，你需要把我的修改同步到你的分支，避免后续合并冲突：
```bash
# 先切回主分支，拉取最新代码
git checkout main
git pull origin main

# 切回你的分支，合并主分支的最新代码
git checkout login-张三
git merge main

# 如果提示「冲突」，看下面的「冲突解决」部分
```

## 四、代码写完后：合并到主分支
### 方式1：我来帮你合并（新手推荐）
你推送分支后，告诉我「我写完了，分支名是 login-张三」，我会在 GitHub/GitLab 上审核代码，然后合并到主分支。

### 方式2：自己提合并请求（可选）
如果想自己操作，打开仓库的 GitHub 页面：
1. 点击顶部的「Pull requests」→「New pull request」；
2. 左边「base: main」（目标分支选主分支），右边「compare: login-张三」（选你的分支）；
3. 点击「Create pull request」，填写备注（比如「张三：完成登录功能开发」）；
4. 通知我审核，我确认后点击「Merge pull request」，你的代码就合并到主分支了。

## 五、常见问题：冲突解决（新手别怕！）
如果合并时提示「冲突」，说明你和我改了同一个文件的同一行代码，按以下步骤解决：
### 1. 打开冲突文件
文件里会有这样的标记：
```
<<<<<<< HEAD
我改的内容
=======
你改的内容
>>>>>>> login-张三
```
### 2. 手动修改冲突
删掉这些标记，保留正确的内容（比如和我沟通后，决定保留你的修改，就删掉我的内容和标记）。

### 3. 重新提交
修改完保存，然后在命令行输入：
```bash
git add .
git commit -m "解决登录页面的代码冲突"
git push
```

## 六、其他常用小操作
### 1. 查看当前在哪个分支
```bash
git branch
```
### 2. 临时保存未完成的修改（比如突然要切分支）
```bash
# 保存修改
git stash save "未完成的登录功能"

# 恢复修改（切回你的分支后）
git stash pop
```
### 3. 删错文件想恢复（未提交的情况下）
```bash
# 恢复某个文件（比如 login.py）
git checkout -- login.py

# 恢复所有文件
git checkout -- .
```

## 总结
1. **核心流程**：拉取最新主分支 → 建自己的分支 → 写代码 → 提交 → 推送 → 合并（解决冲突）；
2. **关键原则**：永远不在 main 分支直接写代码，每次写代码前先 `git pull origin main` 同步最新代码；
3. **冲突不可怕**：冲突只是 Git 提示「有重复修改」，和我沟通后手动改完提交即可。
