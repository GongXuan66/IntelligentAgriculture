# 智慧农业系统常见问题 FAQ

## 设备控制类

### Q1: 如何开启检测点的风机？

要开启风机，您需要知道风机的设备编码。设备编码通常是 `FAN-XXX` 的格式，其中 `XXX` 是编号。使用 `control_device` 工具，传入设备编码和 `on` 命令即可开启。

### Q2: 设备开关命令无效怎么办？

请检查以下几点：
1. 设备编码是否正确（大小写敏感）
2. 命令是否正确（只支持 `on` 或 `off`）
3. 设备是否在线
4. 网络连接是否正常

### Q3: 可以同时控制多个设备吗？

目前系统支持逐个设备控制。如需批量控制，可以循环调用控制接口。

## 环境监测类

### Q4: 如何查看当前温度？

使用 `get_current_environment` 工具，传入检测点 ID（point_id）即可获取该检测点的当前环境数据，包括温度、湿度、光照等。

### Q5: 历史数据可以查询吗？

可以，使用 `get_history_environment` 工具可以查询指定检测点的历史环境数据，支持指定查询条数。

### Q6: 环境数据更新频率是多少？

系统默认每分钟更新一次环境数据。如果需要更高频率，请联系系统管理员。

## 灌溉控制类

### Q7: 如何开始灌溉？

使用 `start_irrigation` 工具，需要传入：
- `point_id`：检测点 ID
- `duration`：灌溉时长（秒），默认60秒
- `mode`：灌溉模式，默认1（滴灌）

### Q8: 如何停止正在进行的灌溉？

使用 `stop_irrigation` 工具，传入灌溉日志 ID（log_id）即可停止。

### Q9: 什么是灌溉需求检查？

`check_irrigation_need` 工具可以根据土壤湿度自动判断是否需要灌溉。当土壤湿度低于阈值时，会返回需要灌溉的建议。

## 报警管理类

### Q10: 如何查看未处理的报警？

使用 `get_unprocessed_alarms` 工具可以获取所有未处理的报警列表。

### Q11: 如何处理报警？

使用 `handle_alarm` 工具，传入报警 ID 和处理备注即可处理报警。

## 系统使用类

### Q12: 会话 ID 是什么？

会话 ID 用于标识用户的对话会话。相同会话 ID 的对话会被串联起来，AI 可以理解对话上下文。如果不指定，系统会使用默认会话。

### Q13: 如何清除对话历史？

使用 `/api/chat/clear` 接口，传入会话 ID 即可清除该会话的对话历史。

### Q14: 支持流式输出吗？

是的，使用 `/api/chat` 接口可以获取流式响应，提供更好的交互体验。
