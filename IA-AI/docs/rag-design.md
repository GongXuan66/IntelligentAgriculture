### 2.1 核心框架

| 组件 | 技术选型 | 版本 | 说明 |

## 4. 数据流设计（从文档到检索的完整流程）

### 4.1 文档处理流程（离线）

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  原始文档   │────▶│   文档加载   │────▶│  文本分块   │────▶│   向量化    │
│  (PDF/MD)   │     │  (Loader)   │     │  (Splitter) │     │ (Embedding) │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                                                                  │
                                                                  ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   应用      │◀────│  Agent 调用  │◀────│ 上下文组装  │◀────│  索引存储   │
│             │     │             │     │ (Context)   │     │ (ChromaDB)  │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

#### 步骤详解

1. **文档加载 (Document Loading)**
   - 支持 PDF、Markdown、TXT、HTML 等格式
   - 使用 LangChain 的统一加载器接口
   - 提取文本内容 + 元数据（来源、标题、创建时间等）

2. **文本分块 (Text Splitting)**
   - 使用 `RecursiveCharacterTextSplitter`
   - 默认块大小: 500 字符
   - 重叠大小: 50 字符（保持上下文连贯）
   - 保留元数据传递

3. **向量化 (Embedding)**
   - 使用 `sentence-transformers/all-MiniLM-L6-v2`
   - 输出 384 维向量
   - 支持中文和英文

4. **索引存储 (Index Storage)**
   - 存储到 ChromaDB
   - 按 collection 隔离不同类型的文档
   - 支持持久化到本地磁盘

### 4.2 检索流程（在线）

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  用户查询   │────▶│  查询预处理  │────▶│   混合检索  │────▶│  结果重排   │
│             │     │             │     │             │     │  (Rerank)   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                                                                  │
                                                                  ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   LLM 回复  │◀────│  Prompt 组装 │◀────│ 上下文筛选  │◀────│  结果过滤   │
│             │     │             │     │             │     │             │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

#### 步骤详解

1. **查询预处理**
   - 关键词提取
   - 查询扩展（可选）
   - 领域识别

2. **混合检索**
   - 向量相似度检索（语义匹配）
   - BM25 关键词检索（精确匹配）
   - 结果合并与去重

3. **结果重排 (Rerank)**
   - 使用交叉编码器重排
   - 提升 top-k 结果的相关性

4. **上下文组装**
   - 选取 top-k 文档片段
   - 组装为 LLM 可用的上下文
   - 添加引用标注

### 5.2 集成方式

#### 方案 A: 并行检索 + 结果合并（推荐）

```python
# 伪代码示例
async def chat_with_rag(message: str, history: list):
    # 1. 意图判断
    intent = classify_intent(message)
    
    # 2. 并行执行
    tasks = []
    if intent.needs_rag:
        tasks.append(rag_retriever.search(message))
    if intent.needs_tool:
        tasks.append(agent.execute_tools(message))
    
    results = await asyncio.gather(*tasks)
    
    # 3. 合并结果
    context = merge_rag_and_tool_results(results)
    
    # 4. LLM 生成
    response = await llm.generate(message, context, history)
    return response
```

#### 方案 B: RAG 作为 Tool

将 RAG 检索封装为 LangChain Tool，Agent 可按需调用：

```python
from langchain_core.tools import tool

@tool(description="""
根据用户问题检索相关文档知识。
适用于：操作指南、故障排查、功能说明、技术细节等问题的解答。
""")
async def search_knowledge_base(query: str) -> str:
    """知识库检索工具"""
    results = await rag_retriever.search(query)
    return format_retrieval_results(results)
```
