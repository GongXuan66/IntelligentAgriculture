# 智慧农业控制系统 - 系统设计文档

## 一、项目概述

### 1.1 项目背景

基于OpenHarmony操作系统，设计开发一款智慧农业控制APP，实现远程监测农作物环境的温度、湿度、光照、二氧化碳，控制水阀、水泵、LED等设备。

### 1.2 系统定位

- **SaaS云服务模式**：公司提供云服务器，农户只需购买网关设备
- **多租户架构**：支持多农户、多农场、多地块
- **智能化决策**：预测性灌溉 + 作物生长阶段感知 + AI助手

### 2.2 硬件层（RK2206开发板）

**开发板能力：**

| 项目 | 说明 |
|------|------|
| CPU | ARM Cortex-M4 @ 200MHz |
| 内存 | 256KB-512KB SRAM |
| 系统 | OpenHarmony LiteOS |
| 语言 | C语言 |
| 网络 | WiFi / 以太网 |

**开发板职责：**
- 采集传感器数据（温度、湿度、土壤湿度等）
- 控制执行器（水泵、LED、风扇等）
- 上报数据到云端服务器
- 接收云端指令执行控制

### 2.3 三层架构

| 层级 | 组件 | 职责 |
|------|------|------|
| **云端** | 云服务器 + Spring Boot + MySQL | 业务逻辑、数据存储、智能算法、用户管理 |
| **边缘** | RK2206 开发板 (LiteOS) | 数据采集、设备控制、上报数据、接收指令 |
| **终端** | OpenHarmony APP (手机) | 远程监控、手动控制、配置管理、AI对话 |

## 四、智能灌溉算法

### 4.1 三层算法架构

```
┌─────────────────────────────────────────────────────────────┐
│                 Layer 3: 预测性灌溉                          │
│              (时机决策 - 决定何时灌溉)                        │
│                  EWMA 时序预测算法                           │
├─────────────────────────────────────────────────────────────┤
│                 Layer 2: 作物生长阶段感知                    │
│              (基础策略 - 确定目标湿度范围)                    │
│                  知识库 + 规则引擎                           │
├─────────────────────────────────────────────────────────────┤
│                 Layer 1: 环境因子调整                        │
│              (参数优化 - 根据环境调整)                        │
│                  温度/湿度/光照因子                          │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 决策类型

| 类型 | 触发条件 | 说明 |
|------|----------|------|
| `predictive` | 预测未来湿度 < 阈值 | 预防性灌溉，避免水分胁迫 |
| `adaptive` | 当前湿度 < 阈值 | 立即灌溉 |
| `stage_based` | 有作物配置，无需灌溉 | 基于生长阶段 |
| `manual` | 用户手动触发 | 手动灌溉 |

### 4.3 作物生长阶段知识库

系统预置了10种常见作物的生长阶段配置：

| 作物 | 阶段数量 | 特点 |
|------|----------|------|
| 番茄 | 5阶段 | 播种→幼苗→开花→结果→成熟 |
| 黄瓜 | 4阶段 | 喜湿作物，需水量大 |
| 辣椒 | 4阶段 | 开花期敏感 |
| 茄子 | 4阶段 | 结果期需水量大 |
| 水稻 | 5阶段 | 水生作物，湿度要求高 |
| 玉米 | 5阶段 | 抽穗期为需水关键期 |
| 蔬菜(通用) | 3阶段 | 通用配置 |
| 草莓 | 4阶段 | 收获期控水提质 |
| 西瓜 | 5阶段 | 成熟期控水提高糖度 |
| 白菜 | 4阶段 | 结球期需水量大 |

## 五、AI智能助手（创新功能）

### 5.1 功能概述

| 功能 | 表 | 说明 |
|------|-----|------|
| **AI对话** | `chat_history` | 农户主动咨询种植问题 |
| **AI推荐** | `ai_recommendation` | 系统主动推送智能建议 |
| **智能报告** | `ai_analysis_report` | 自动生成日报/周报 |

### 5.2 AI对话 vs AI推荐

| 对比项 | AI对话 | AI推荐 |
|--------|--------|--------|
| 触发方式 | 农户主动问 | 系统主动推送 |
| 场景 | "番茄为什么黄叶？" | 系统发现异常，推送建议 |
| 时机 | 用户使用时 | 后台分析后主动提醒 |

### 5.3 AI推荐类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `irrigation` | 灌溉建议 | "明天高温，建议今天傍晚灌溉" |
| `planting` | 种植/采摘提醒 | "草莓进入最佳采摘期" |
| `pest` | 病虫害预警 | "晚疫病发生风险较高" |
| `alert` | 设备/环境警报 | "温度过高，已超阈值" |

### 5.4 为什么AI推荐需要存数据库

1. **持久化** - 推荐内容不会丢失，用户随时可查看
2. **状态追踪** - 记录是否采纳/忽略
3. **历史追溯** - 可查看历史推荐记录
4. **避免重复** - 不重复推送同类推荐
5. **效果统计** - 分析推荐准确率

## 六、数据隔离

### 6.1 多租户查询示例

```sql
-- 查询用户的所有检测点
SELECT mp.* 
FROM monitor_point mp
JOIN farm f ON mp.farm_id = f.id
WHERE f.user_id = ?;

-- 查询用户的所有传感器数据
SELECT sd.* 
FROM sensor_data sd
JOIN monitor_point mp ON sd.point_id = mp.id
JOIN farm f ON mp.farm_id = f.id
WHERE f.user_id = ?;
```

### 6.2 权限控制

- 农户只能查看和操作自己农场的数据
- 管理员可以查看所有数据
- 所有API接口需要验证用户身份

## 九、比赛创新亮点

### 9.1 技术亮点

1. **预测性灌溉** - EWMA算法预测未来湿度，实现预防性灌溉
2. **作物生长阶段感知** - 根据作物生长周期动态调整灌溉策略
3. **AI智能助手** - 对话式咨询 + 主动推荐 + 智能报告

### 9.2 架构亮点

1. **云边端协同** - 云服务器 + 边缘网关 + 移动终端
2. **多租户SaaS** - 支持多农户、多农场、多地块
3. **国产化** - 基于OpenHarmony生态

### 9.3 答辩要点

```
"我们的智慧农业系统采用三层架构："

云服务器层：
├── 多租户管理，数据隔离
├── 智能灌溉算法（预测+阶段感知）
└── AI智能助手（对话+推荐+报告）

边缘网关层（RK2206 + OpenHarmony LiteOS）：
├── 数据采集与上报
├── 设备控制执行
└── 本地缓存保障

移动终端层（OpenHarmony APP）：
├── 实时监控
├── 远程控制
└── AI交互
```
